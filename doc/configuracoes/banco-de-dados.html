Bom, se a sua aplicação não requer banco de dados, você não irá precisar fazer nada em relação ao banco de dados, pois o Zeanwork esta pronto para ser executado. Mas se você irá utilizar banco de dados devemos fazer as configurações para o mesmo.
Para definir a configuração, você deve ir até o arquivo App/Configs/database.php e localizar a configuração chamada database.

Veja exemplo:
Configure::write('database', array(
	      'development' => array(
				  'persistent' => false //Indica se a conexão será persistente.
				, 'drive' => 'MySql' //Tipo do banco de dados.
				, 'host' => 'localhost' //Endereço do servidor de banco de dados.
				, 'user' => 'root' //Usuário para conexão com o banco de dados.
				, 'password' => '' //Senha para conexão com o banco de dados.
				, 'database' => 'zeanwork' //Nome da tabela do banco.
				, 'prefix' => null //Prefixo nos nomes das tabelas do banco de dados
				, 'charset' => 'utf8' //Charser do banco de dados
		       )
	      )
	  );


Auto conectar-se com o banco de dados

Para definir se o zeanwork conectará automáticamente com o servidor de banco de dados, você deve ir até o arquivo App/Configs/database.php e localizar a configuração chamada auto.connect.database.
O valor a ser definido é boolean (true/false).

Veja exemplo:
Configure::write('auto.connect.database', true);


Auto desconectar-se com o banco de dados

Para definir se o zeanwork desconectará automáticamente com o servidor de banco de dados, você deve ir até o arquivo App/Configs/database.php e localizar a configuração chamada auto.disconnect.database.
O valor a ser definido é boolean (true/false).

Veja exemplo:
Configure::write('auto.disconnect.database', true);

Nota: Se a configuração de conexão com o banco de dados for persistente, então será ignorado esta configuração.
